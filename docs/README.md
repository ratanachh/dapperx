# Building the docs site

This site is built with [DocFX](https://dotnet.github.io/docfx/): conceptual pages under `articles/` are
hand-written Markdown, and `api/` is generated from the XML doc comments in `src/DapperX`.

## Install DocFX

```bash
dotnet tool update -g docfx
```

## Build

```bash
dotnet restore DapperX.slnx
docfx build docs/docfx.json
```

Output goes to `docs/_site` (gitignored — never hand-edit it).

## Preview locally with live reload

```bash
docfx docs/docfx.json --serve
```

Then open `http://localhost:8080`.

## Notes

- `docs/api/*.yml` and `docs/api/toc.yml` are generated by the metadata step and gitignored; only
  `docs/api/index.md` (the section landing page) is hand-written and committed.
- The metadata step only extracts from `src/DapperX/DapperX.csproj` — `DapperX.Generator` is an internal
  Roslyn analyzer with no consumer-facing API and is intentionally excluded.
- Pushing to `main` with changes under `docs/**` or `src/DapperX/**` rebuilds and redeploys the site via
  `.github/workflows/docs.yml`.
